# @zenone/widget-wrangler 🤠

把零散的小组件都圈起来。每个 Pi 扩展都可以通过 `ctx.ui.setWidget(...)` 在编辑器上方或下方绘制一个小组件。装了几个扩展后，屏幕很快就会变得杂乱。**小组件管理（Widget Wrangler）** 把所有扩展的所有小组件都收进一个面板，你可以逐个切换**显示**或**隐藏**。

它完全通用：不关心你运行哪些扩展。任何调用 `setWidget` 的扩展都会自动出现在围栏中。

## 功能

- 拦截 `ctx.ui.setWidget`（所有扩展共享的 UI 接口），以小组件 id 为键维护一个实时注册表。
- 面板（`/wrangle` 或 `Ctrl+Shift+W`）列出所有已发现的小组件，每个带 **显示**/**隐藏** 开关——体验与 Pi 的 `/mcp` 面板相同。
- 隐藏的小组件在到达屏幕之前就被抑制。你的选择会**全局**持久化（保存到 `~/.pi/agent/widget-wrangler.json`），所以跨会话、重载和分支，相同的小组件都保持隐藏。一次设置，全局生效。
- 它不会干扰 `ctx.ui.notify`，因此扩展的错误/信息通知仍然会正常弹出并自动消失——即使对于已隐藏的小组件也是如此。

## 用法

- `/wrangle` — 打开围栏面板。
- `Ctrl+Shift+W` — 打开围栏面板（默认快捷键，可配置）。
- 在面板中：方向键移动，`空格`/`回车`切换，输入文字搜索，`Esc`关闭。修改即时生效。

### 修改快捷键

快捷键通过 CLI 标志 `widget-wrangler-key` 配置。启动 pi 时覆盖：

```bash
pi --widget-wrangler-key=alt+w
```

传入空值可完全禁用快捷键，仅使用 `/wrangle` 命令：

```bash
pi --widget-wrangler-key=
```

> 关于 `Alt`/`Option`：`alt+*` 快捷键在 Linux 上可靠，但在 macOS 上取决于终端配置（例如 iTerm2 需要 "Use ⌥ as Meta"）。默认的 `ctrl+shift+g` 是跨 macOS、Linux 和 Windows 最通用的选择。

## 安装

将其添加到你的 Pi 扩展中（npm 包或本地文件）。它会注册一个命令和一个快捷键；无需其他配置。

## 工作原理（以及一个注意事项）

所有扩展共享同一个 `ctx.ui` 对象，因此只包装一次 `setWidget` 就能拦截所有扩展的小组件调用。小组件第一次出现在围栏中的时机是它的所有者扩展第一次设置它。大多数状态类小组件每轮对话都会重新设置，因此它们几乎立即出现；只设置一次且在本次扩展加载之前就已设置的小组件，需要等到下一次更新后才能被控制。
